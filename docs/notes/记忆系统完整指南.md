## 一、记忆类型总览

Mirror 采用多层记忆架构，所有用户记忆最终汇聚到 **CoreMemory** 根容器中。

### 1.1 CoreMemory 顶级结构

```
CoreMemory (根容器)
├── self_cognition    # Agent 自我认知
├── world_model       # 用户/世界知识
├── personality       # 人格状态
└── task_experience   # 任务经验
```

**文件位置**: `app/memory/core_memory.py:282`

### 1.2 各类记忆详解

| 记忆类型 | 类名 | 所在位置 | 说明 |
|---------|------|---------|------|
| **自我认知** | `SelfCognition` | `:233` | Agent 理解自身能力/限制 |
| **世界模型** | `WorldModel` | `:244` | 用户事实、推断、关系 |
| **人格状态** | `PersonalityState` | `:259` | 性格、关系风格、会话适应 |
| **任务经验** | `TaskExperience` | `:273` | 压缩的任务学习摘要 |
| **确认事实** | `FactualMemory` | `:65` | 用户明确确认的知识 |
| **推断记忆** | `InferredMemory` | `:72` | AI 推断但未确认 |
| **关系记忆** | `RelationshipMemory` | `:79` | 主体-关系-客体 三元组 |
| **结构化教训** | `Lesson` | `app/tasks/models.py:45` | 从对话/任务提取的教训 |

### 1.3 SelfCognition 内部结构

```python

├── capability_map     # {domain: CapabilityEntry}  能力列表
```

### 1.4 WorldModel 内部结构

```python

├── confirmed_facts        # [FactualMemory]      已确认事实

```

## 三、记忆触发机制

### 3.1 触发事件总表

| 事件类型 | 触发时机 | 处理器 |
|---------|---------|-------|
| `dialogue_ended` | 对话结束 | `SignalExtractor`, `ObserverEngine`, `ProactivityService` |
| `lesson_generated` | 生成教训 | `CognitionUpdater` |
| `task_completed` | 任务完成 | `MetaCognitionReflector` |
| `task_failed` | 任务失败 | `MetaCognitionReflector` |
| `hitl_feedback` | 人工确认反馈 | `CognitionUpdater`, `PersonalityEvolver` |

### 3.2 SignalExtractor - 对话信号提取

**文件**: `app/evolution/signal_extractor.py`

#### 会话级适应（不持久化）

| 信号类型 | 触发关键词 | 存储位置 |
|---------|-----------|---------|
| `prefer_concise` | "简洁一点", "简短", "concise", "shorter" | SessionAdaptation |
| `language_zh` | "中文", "说中文", "chinese" | SessionAdaptation |
| `tone_direct` | "少点客套", "直接一点" | SessionAdaptation |

#### 支持偏好（持久化）

触发关键词：
- **Listening 模式**: `"just listen"`, `"don't give advice"`, `"先听我说"`, `"不要急着给建议"`
- **Problem-solving 模式**: `"help me solve"`, `"tell me what to do"`, `"直接告诉我怎么做"`, `"帮我解决"`

生成 `Lesson` → 存入 `world_model.confirmed_facts` 或 `inferred_memories`

#### 主动性偏好（持久化）

触发关键词：
- **允许跟进**: `"check in later"`, `"follow up later"`, `"feel free to check in"`
- **禁止提醒**: `"don't remind me"`, `"don't follow up"`, `"no reminders"`

生成 `Lesson` → 存入 `world_model.proactivity_state.latest_preference_override`

### 3.3 ObserverEngine - 观察引擎

**文件**: `app/evolution/observer.py`

- **触发条件**: 30秒窗口 或 累积5条消息
- **提取方式**: LLM 提取三元组
- **关系类型**: `PREFERS`, `DISLIKES`, `USES`, `KNOWS`, `HAS_CONSTRAINT`, `IS_GOOD_AT`, `IS_WEAK_AT`
- **存储位置**: GraphStore (Neo4j) + VectorRetriever (Qdrant)

### 3.4 MetaCognitionReflector - 元认知反思

**文件**: `app/evolution/reflector.py`

- **触发条件**: `task_completed` 或 `task_failed` 事件
- **前提条件**: 任务存在于 task_store，且 lesson.confidence >= 0.5
- **LLM 提取字段**: `root_cause`, `lesson`, `is_agent_capability_issue`, `subject`, `relation`, `object`, `confidence`

### 3.5 CognitionUpdater - 认知更新器

**文件**: `app/evolution/cognition_updater.py`

处理 `lesson_generated` 事件，将 Lesson 分类并写入对应记忆：

```
lesson.is_agent_capability_issue = True
    └── 更新 SelfCognition.capability_map
    └── 增加/减少对应 domain 的 confidence

lesson.is_agent_capability_issue = False
    ├── domain = "support_preference"
    │       └── FactualMemory 或 InferredMemory
    ├── domain = "proactivity_preference"
    │       └── FactualMemory 或 InferredMemory
    ├── 有 subject + relation + object
    │       └── RelationshipMemory (存 Neo4j)
    └── 其他情况
            └── InferredMemory (默认)
```

## 七、当前限制与已知问题

### 7.1 为什么"我喜欢写代码"没有触发记忆？

**原因**: SignalExtractor 只识别预定义的关键词模式。

当前会触发记忆的关键词：
```
支持偏好: "just listen", "don't give advice", "先听我说", "不要急着给建议"
主动性:   "don't remind me", "check in later", "follow up later"
会话适应: "简洁一点", "说中文", "少点客套" (仅当前会话)
```

**通用事实**（如"我喜欢写代码"、"我在北京工作"）目前**不会被自动提取**。

### 7.2 ObserverEngine 的关系类型限制

当前只支持固定的关系类型：
- `PREFERS`, `DISLIKES`, `USES`, `KNOWS`
- `HAS_CONSTRAINT`, `IS_GOOD_AT`, `IS_WEAK_AT`

不支持任意关系的自由提取。

### 7.3 HITL 确认机制

以下情况需要人工确认：
- `sensitivity == "sensitive"`
- `confidence < 0.75`
- `confirmed_by_user == False`
